# プログラミング基礎知識の日本語での再翻訳メモ

この文書は、raw に残っていた思考整理メモを、現時点で確認できる user 側方針に寄せて整理し直した作業用版である。  
既存訳語の違和感と再翻訳の必要性は残す。  
比喩や歴史説明を使う場合も、中心 doctrine に固定せず、候補や保留として扱う。

## この文書の狙い

- プログラミング関連語が、日本語話者にとって「自然に理解できる概念」になっていない問題を見直す
- 既存訳語をそのまま受け入れるのではなく、原語の役割やニュアンスを再検討する
- 日本語でプログラムを考え、説明し、応用するための足場を整える

## この文書で避ける前提

- `陰陽術` を翻訳体系の中心 doctrine として固定しない
- `思想・実装・祈り` や `思想・祈り・実装` のような定型を採用しない
- 性別、善悪、光と闇などの属性づけを `陰陽` に持ち込まない
- `祈り` や `呪術` を、プログラミング理解の中心 doctrine として固定しない

## 再翻訳の必要性

- プログラミング用語の多くは、英語圏の生活感覚や設計文化を前提にしている
- 日本語では、既存訳語が数学寄り、制度寄り、抽象寄りに固まりすぎて、日常的なニュアンスが抜けることがある
- その結果、用語が「呪文」のように見え、何のための語なのかがつかみにくくなる
- 再翻訳の目的は、日本語話者が英語の語を丸暗記することではなく、日本語でも役割と場面を考えられるようにすることにある

## 再翻訳の基本姿勢

- 訳語は固定対応表で決めず、文脈の中でその語が果たしている役割を優先する
- 原文のニュアンスを壊さないことを重視する
- 候補語は候補として扱い、十分な根拠がない段階で canonical term にしない
- 原語の語源や使われ方を確かめる姿勢は保つが、思弁をそのまま事実扱いしない

## 再検討したい語

### variable

- `変数` という訳は残せるが、「数」に限定される印象を与えやすい
- 文脈によっては、「変わりうる状態」や「状態を入れておくもの」として説明した方が分かりやすい可能性がある

### function

- `関数` は数学の文脈では適切だが、プログラミング入門では硬すぎる場合がある
- 文脈によっては、「働き」「機能」「ふるまい」といった説明語の方が役割を掴みやすい

### object

- `オブジェクト` をそのまま残すだけでは、何を指すか見えにくい
- 少なくとも一部の文脈では、「振る舞いを持つ単位」「やり取りできる存在」としての説明を検討する余地がある

### class

- `クラス` は学級や分類の印象を引きずりやすい
- `類` と漢語で寄せるよりも、「たぐい」と読む感覚の方がしっくりくる可能性がある
- ただし、分類、ひな型、生成規則のどこを強調したいかは source ごとに判断が必要

### statement

- `文` では自然言語の文と混同しやすい
- 少なくとも、安易に「ひとまとまりの処理」「一つの手続き」と言い換えるのは危うい
- source ごとに、`文`、`記述`、`一つの述定単位` のどれとして説明するのがよいかを詰める必要がある

### computer

- 原義に「計算する人」のニュアンスがある点は検討材料として重要
- ただし、`演算体` や `補助者` などの候補語は、現時点では作業上の仮案に留める
- 既存訳 `電子計算機` を全面否定するより、source ごとにどこまで説明を補うかを考える方が安全

### cyber

- `舵取り` や `操る` は説明補助として有効
- ただし、神秘化や比喩の膨張ではなく、制御・接続・調整の話として扱う

## 保留してよい比喩

### 願いの媒介としての機械

- 人の願いを言葉やコードに託し、機械に動作させる、という見方自体は日本語的な感覚へ落とし込む一例として検討の余地がある
- `願掛け` や `呪術` を引き合いに出す書き方も、一例としては残しうる
- ただし、これを AI やプログラミングの本質説明として固定しない
- この比喩を使う場合も、操作、実装、手順との接続を切らない

### command / order / instruction / goal / evaluation

- これらは一律の日本語へ固定しない
- 文脈によっては `命令` より `お願い` や `指示` が近いこともある
- 文脈によっては `目標` より `望み` が近いこともある
- 文脈によっては `評価` より `確かめる` が近いこともある
- ただし、どれも例であって、原文のレイヤーと場面を優先して判断する

## この段階で保留にする論点

- Turing / Boole / Shannon の系譜をどう並べるか
- `computer` の日本語候補を何に固定するか
- `気` を翻訳補助語としてどこまで使うか
- 東洋思想との接続を別章として立てるか
- `陰陽` を補助比喩としてどこまで出すか
- `願いの媒介としての機械` や `願掛け / 呪術` の比喩を、どの程度まで本文に残すか

## いま残してよい核

- 日本語話者にとって、既存訳語が役割やニュアンスを伝えきれていない箇所がある
- 再翻訳は、英語の語を日本語話者が自分の言葉で考え直すために必要
- その際に重要なのは、固定の言い換え表ではなく、原文の場面・語り口・役割を読む姿勢である
