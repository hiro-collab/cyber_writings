# Workflow

この文書は、ChatGPT履歴や断片的なメモを、公開可能な文章へ育てるための作業手順である。

## 0. 生データを置く

ChatGPTのエクスポート、医療記録、未整理の長いログは、原則としてGitに載せない。

推奨置き場:

- `99_private/`
- `raw_exports/`
- `exports/`

これらは `.gitignore` で除外している。GitHubにアップロードする前提では、生データをそのまま公開しない。

## 1. 断片化する

`00_fragments` に、後で使える単位で抜き出す。

ChatGPTエクスポートを手動で読む場合は、[tools/chatgpt_export_viewer.html](../tools/chatgpt_export_viewer.html) を使う。
`conversations.json` をブラウザで読み込み、自分の入力とChatGPTの出力を分けて確認できる。

抜き出す単位の例:

- 症状の変化がわかる発言
- その時点での自己理解
- 脳機能、認知、記憶、言語、注意に関する考察
- サイバー、情報環境、身体、自己像をつなぐ発想
- あとで誤解されそうだが重要な迷い

断片には、可能なら日付、出典、確信度、公開可否を付ける。

## 2. 整理する

`01_curated` では、断片を時系列や主題ごとにまとめる。

この段階では、まだ完成した文章にしなくてよい。重要なのは、次を分けること。

- 実際に起きたこと
- 当時そう感じたこと
- 後から考えた解釈
- まだ仮説にすぎないもの
- 調査や出典確認が必要なもの

## 3. 原稿化する

`02_manuscript` では、読者に届く順番へ組み替える。

原稿化では、記録の完全性よりも読み筋を優先する。ただし、出来事の順序や医学的・技術的な主張は雑に補わない。

## 4. 公開前確認

公開前に最低限確認すること:

- 他者の個人情報が含まれていないか
- 自分の住所、職場、連絡先、アカウント、診療先などが露出していないか
- 医学的事実と自己解釈が混ざっていないか
- 特定人物への断定的な非難になっていないか
- 収益化する文章として、無料公開部分と有料部分の境界があるか

## Codexとの作業単位

Codexに依頼しやすい単位:

- 「このChatGPT断片から症状記録だけを抜き出して」
- 「事実、感情、仮説、調査要の4分類にして」
- 「このノートをエッセイの見出し案にして」
- 「意味を変えずに読みやすくして」
- 「公開前に個人情報と断定表現をチェックして」
