## 最初に公開する記事案

### タイトル

### 構成案

```markdown
# function を「関数」とだけ訳すと、何が見えなくなるのか

## 1. 関数という訳は間違いではない

数学では function は関数でよい。
しかしプログラミング入門では、それだけでは分かりにくいことがある。

## 2. 英語の function には「働き」の感覚がある

function は、何かが果たす役割、働き、機能を指す。
プログラムの中では、複数の処理をまとめ、名前をつけ、必要なときに呼び出せるようにしたものでもある。

## 3. 「命令の集まり」より「お願いが働きになる」と見る

人がコンピューターにしてほしいことを書く。
それを順番に並べる。
まとまりにする。
名前をつける。
必要なときに呼ぶ。

そう考えると、function は「お願いの束が、呼び出せる働きになったもの」と説明できる。

## 4. 訳語を固定しない

だから function を常に「働き」と訳せばよい、という話ではない。
重要なのは、文脈に応じて、原文で何が立ち上がっているかを見ることである。

## 5. 日本語で考え直す意味

技術用語を日本語で考え直すことは、ガラパゴス化ではない。
むしろ、技術を自分たちの生活感覚と接続し直す作業である。
```
